About this

Weather App

Front-End Web Developer, based in Portugal
Purpose

This web app allows users to obtain live weather data (via API calls) for a desired location or for the user's position via geolocation. It also features a 5-day forecast, and changes color depending on whether it is daytime or nighttime in the location of interest.

As a fun addition, I coded a couple of festive themes for:

  • Christmas and New Year (December and January)
  • Easter (April)
  • Halloween (October)
If using this app on a smart phone or small tablet, I recommend rotating the screen.

Construction

I wrote this code using HTML, CSS and JavaScript in order to create an aesthetically pleasing, functional weather application. I began by creating a wireframe design on Invision, I then translated that into basic HTML, along with some CSS. For my editor, I originally began on CodeSandBox, but transitioned across to Visual Studio Code, which was tracked by my GitHub.

I then moved on to JavaScript where I created:

  • A location search engine with submit button
  • Integrated Axios and APIs via OpenWeatherMap to obtain live weather data
  • Integrated Axios and APIs to obtain a 5-day forecast
  • Implemented geolocation in the "Current Location" button, if permitted by the user
  • Use of of APIs to establish the desired weather condition codes, and display the correspoding weather icons
  • Creation of unit conversion buttons for 째C and 째F
Challenges and Solutions

This project was a fulfilling challenge which allowed me to not only present the skills I had learned but also push myself further in tackling problems I hadn't encountered at that point in my education.

Example 1:

A simple example would be that long city names, e.g. Copenhagen would be displayed in the h1, and badly overlap with other elements of the app.

I tackled this by using JavaScript and CSS, so that if the city name consisted of 8 or more characters, then the font-size would reduce by 20px. This allowed for a more pleasant User Experience, and resolved the bug.

Example 2:

Another example involved the idea of re-using the same button to enable and disable the festive themes I designed. I understood that it would be different to having 2 separate 째C and 째F buttons, as they both had their own EventListener in JavaScript.

Despite this being something I hadn't learned yet, I was determined to find a way to achieve it using my problem-solving and reasoning skills.

I reached my solution via a combination of JavaScript expressions:

  • Creating a parent function displayThemeButton, with multiple child functions for each theme, such as displayChristmas() and removeChristmas()
  • Staggering 1 EventListener for displayChristmas() and 1 for removeChristmas()
  • Use of if else statements to ensure that the Christmas Theme button would appear if the current month is December or January
  • Targetting the button element by class using document.querySelector
  • Assigning a new id to the element using setAttribute
  • Implementation of EventListener code which awaited a click to display, and a further click to remove the theme
  • Use of classList.add and classList.remove to change background theme from day/night mode to the relevant theme
